# SAP P2P Capstone Project (Simulation-Based)

## 📌 Project Title
Procure-to-Pay (P2P) Cycle in SAP MM using ABAP Logic Simulation

---

## 📖 Project Overview
This project presents a simulation of the Procure-to-Pay (P2P) cycle in SAP Materials Management (MM) using ABAP programming concepts. Due to the unavailability of a live SAP system, the entire implementation is designed as a logical prototype using structured data and ABAP-style code written in text format.

The project demonstrates how procurement processes can be modeled programmatically without actual SAP execution.

---

## 🔄 P2P Process Covered
The simulation includes all major stages of the procurement lifecycle:

- Purchase Requisition (PR)
- Purchase Order (PO)
- Goods Receipt (GR)
- Invoice Verification
- Payment Status Tracking

---

## 🧩 Modules Simulated
- Vendor Master Simulation  
- Material Master Simulation  
- Purchase Requisition Simulation  
- Purchase Order Simulation  
- Goods Receipt Simulation  
- Invoice Simulation  
- Payment Tracking Simulation  

---

## ⚙️ Implementation Approach
The project is implemented using:

- ABAP-style syntax (written in `.txt` files)
- Internal table logic (simulated)
- Structured programming concepts
- Modular design for each P2P stage

No real SAP system, database, or runtime execution is used.

---

## 📊 Key Feature
### ZP2P_TRACKING_REPORT (Simulated)

A custom report is designed conceptually to integrate all stages and display:

- Vendor Details  
- Material Details  
- PR, PO, GR, Invoice Status  
- Payment Status
